# Sky Omega 1.8.1 — DrHook.Mcp stepping path switched to DrHook.Engine

**Released:** 2026-05-22
**Tag:** `mercury-1.8.1` (commit `ba9d8ca`)
**Predecessor:** [Sky Omega 1.8.0](1.8.0.md) — cognitive-layers entry point; substrate-hardening line closed

## Headline

`DrHook.Mcp` switched from the netcoredbg-DAP-backed `SteppingSessionManager` to the BCL-only `DebugSession` in `DrHook.Engine`. All 17 stepping MCP tools now drive ICorDebug directly via source-gen COM + per-RID `libdbgshim` — no `netcoredbg` subprocess, no DAP over stdio.

This is the first 1.8.x substrate-discipline release. The cognitive-layers framing in 1.8.0 named the DrHook engine BCL-only rewrite as the first substrate-side task; 1.8.1 lands the switchover.

## What 1.8.1 carries

The Phase 3 substrate arc inside `DrHook.Engine` that made the switchover possible. Forty PoC probes (02–32) at write time validated each capability against macOS/ARM64 CoreCLR before any of it was wired into the MCP surface:

| Capability | Substrate work | Probe(s) |
|---|---|---|
| Attach / detach / continue-loop / stopping model | `CallbackPump`, `IManagedCallbackSink`, `ManagedCallbackHost` (source-gen COM CCW), `ICorDebugController` ownership | 02–10 |
| Metadata + breakpoints + stepping + stack frames + variables | `MetadataResolver`, `Breakpoints`, `Stepping`, `Frames`, `Variables` | 11–18 |
| Func-eval (the netcoredbg deadlock isn't a platform limit) | `Eval` interop slots from `cordebug.idl`, verified | 19–21 |
| Conditional breakpoints + general member resolution | Roslyn member-access walker, runtime-type-dispatched getter func-eval | 22–25 |
| Exception breakpoints + func-eval at exception stop | `ICorDebugManagedCallback2::Exception`, `GetCurrentException`, exception-rooted walker operand | 26–27 |
| `BreakpointPolicy` (one type, four configurations) | Conditional / logpoint / hit-count / fault unified; `BoundedLogSink`; tri-state condition outcome with `IsFault` `LogRecord` | 28–30 |
| `DebugSession.Pause` / `Launch` (attach-before-main) | `ICorDebugController.Stop`, `dbgshim` `RegisterForRuntimeStartup` flow | 31, 33 (33 lands in 1.8.2) |
| Persistent exception filter | Arm / List / Remove / Clear over `WaitForStop` | 32, 34 (34 lands in 1.8.2) |

## What 1.8.1 does NOT carry

- The `src/DrHook/` netcoredbg-DAP project is still present on disk in 1.8.1 (retired in 1.8.2).
- `drhook_step_test` is wired but its engine path returns a structured "not yet ported" — the `VSTEST_HOST_DEBUG` testhost-PID-discovery dance from the netcoredbg version is not in DrHook.Engine. ADR-007 (Proposed 2026-05-23) removes the tool entirely in Phase 7 rather than porting it.
- Object-inspection substrate beyond primitive locals (reference-string rendering, field walking, array rendering, subclass-aware exception filter, cross-module member resolution) — those land in 1.8.2.

## Substrate posture at 1.8.1

- `DrHook.Engine` substrate: BCL + P/Invoke + source-gen COM. Two diagnostics dependencies remain per ADR-009: `Microsoft.Diagnostics.NETCore.Client` (managed; admitted) + `libdbgshim` (native asset; bundled per-RID).
- macOS/arm64 the only validated platform.
- 47 `DrHook.Engine.Tests` unit tests pass (CallbackPump, ManagedCallbackHost, ProcessInspector, SymbolReader, BreakpointPolicy, BoundedLogSink).
- Mercury substrate posture unchanged from 1.8.0 — no Mercury changes in this release.

## What's next

1.8.2 closes the substrate-independence ceremony (`src/DrHook/` retired, per-RID `libdbgshim` bundling formalized). ADR-007 (Proposed 2026-05-23, drafted post-1.8.2) sequences the production-suitability work: teardown + concurrency hardening, test-runner debugging without env-flag tricks, integration-test mechanism characterization, cross-platform validation campaign.
