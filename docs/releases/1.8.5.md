# Sky Omega 1.8.5 — DrHook navigable inspection, and the Mercury silent-drop it surfaced

**Released:** 2026-06-22
**Version bump:** `Directory.Build.props` → 1.8.5
**Predecessor:** [Sky Omega 1.8.4](1.8.4.md) — Mercury convergence: one execution path, one expression evaluator

## Headline

1.8.5 is a **dogfooding** release. We rebuilt DrHook's variable inspection — the eager object-graph walk was crashing the engine on real, large frames — into a lazy, navigable model. Then we pointed the fixed DrHook at Mercury to hunt a bug, and it traced a **silent data-loss bug** to a single line: string literals of 1023 characters or more were vanishing from SELECT result rows. We fixed that too. DrHook **found the bug and validated the fix**, in one session, with no bespoke instrumentation. That is the runtime-observation substrate doing exactly what it is for.

## DrHook — lazy, navigable variable inspection (ADR-007 inspection item + ADR-013 D1/D2)

`drhook_locals` used to eagerly walk a stopped frame's object graph to a requested depth. Over a wide live graph (Mercury's `QuadStore` — mmap-backed indexes, `FileStream`/`SafeHandle` internals) a depth-2 walk **crashed the engine with a native `SIGSEGV` in coreclr's unwinder** (`RtlpUnwindFunctionFull`), dropping the MCP server while the debugged target survived. The fault is native — uncatchable by a managed `try/catch` — so a cap/truncation would not have made it safe; it would have made DrHook *blind exactly where it is most needed*. A debugger exists to observe arbitrary real state; the large frame is the point, not the edge.

The fix is architectural: inspection is now **lazy and navigable**. `drhook_locals` returns **one level**, each node flagged `hasChildren`; the new **`drhook_expand`** tool walks deeper **on demand**, one bounded level at a time (by field name or `[i]`), so an arbitrarily large or deep graph stays **fully observable** while the walk is bounded by construction. [ADR-013](../adrs/drhook/ADR-013-value-type-refstruct-inspection.md) D1/D2 then extended expansion to **VALUETYPE** structs/spans and **BYREF** ref-struct receivers (a struct method's `this`), so the zero-GC, span-dense substrate code DrHook is meant to observe opens up. 119/119 `DrHook.Engine` tests.

## Mercury — growable result-binding buffer (ADR-050)

With DrHook working, we hunted a bug it had earlier hinted at, and it traced the loss path to `BindingTable.Bind` (`TriplePatternScan.TryBindVariable` → `Bind`) via the live call stack. The defect:

```csharp
if (_stringOffset + value.Length > _stringBuffer.Length) return;   // silent: no bind, no error, no anomaly
```

The result-binding string buffer was a fixed **1 KB** pooled array, and `Bind` **silently returned** on overflow — so any literal whose lexical form (value + 2 quotes) exceeded 1024 chars, i.e. **≥ 1023 characters**, was dropped from SELECT result rows (the row came back missing the binding). A *read-back* bug, not storage — and it had been quietly corrupting Mercury's own long `ck`/memory writes.

[ADR-050](../adrs/mercury/ADR-050-growable-result-binding-buffer.md) makes the buffer **grow on demand** via `ArrayPool<char>.Shared` (rent larger, copy the live prefix so offsets stay valid; steady state stays zero-GC, growth is pooled). The literal-size policy is now **specified** as the store's `MaxAtomSize` — RDF 1.1 / SPARQL impose no limit and no surveyed provider publishes a small per-literal cap, so the only principled ceiling is Mercury's existing 1 MB atom cap. Verified: `tools/repro-literal.cs` (every length 300–4000 round-trips, was a hard cliff at 1023), `LongLiteralRoundTripTests` (6/6, 500–100,000 chars), full suite **4,699 / 0 failed / 6 skipped**, and **live under DrHook** — a breakpoint at the growth point fired during a SELECT's result materialization over a 1023-char literal.

## The dogfood, and the next gap

The capstone validated both fixes in one DrHook session — and then **surfaced DrHook's next gap**: inspecting a *complex* ref-struct (`BindingTable`, with `Span<Binding>` / `Span<char>` fields) aborts mscordbi (`SIGABRT`; the target survives). Simple structs/spans inspect cleanly; this shape does not, and there is no fault *containment* yet. [ADR-014](../adrs/drhook/ADR-014-inspection-fault-containment.md) (Proposed) scopes value-type read safety + inspection-fault containment — the finding the dogfood earned.
