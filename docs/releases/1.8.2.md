# Sky Omega 1.8.2 ‚ÄĒ netcoredbg retired; DrHook substrate-independence reached

**Released:** 2026-05-23
**Tag:** `mercury-1.8.2` (commit `432e4c8`)
**Predecessor:** [Sky Omega 1.8.1](1.8.1.md) ‚ÄĒ stepping path switched to DrHook.Engine

## Headline

`src/DrHook/` retired entirely. `DrHook.Engine` is now Sky Omega's sole .NET runtime-observation substrate. The substrate-independence bar named in ADR-006 is reached: `DrHook.Core` has zero spawns of `netcoredbg`; `Microsoft.Diagnostics.NETCore.Client` (managed) + `libdbgshim` (native asset) remain as the only diagnostics dependencies, both admitted per ADR-009.

The Phase 3 substrate arc closes with the object-inspection capabilities that the MCP surface needs for AI-coding-agent use against real .NET workloads.

Phase 3 substrate is **complete** at probe 40. Every capability ADR-006 named as load-bearing is green on macOS/arm64.

## Structural retirement

- `src/DrHook/` directory deleted (was the netcoredbg-DAP-backed runtime observation substrate).
- `DrHook.Mcp.csproj` references `..\DrHook.Engine\DrHook.Engine.csproj` exclusively. The bundled native dbgshim assets that were pre-staged in 1.7.x are now live consumers, not dead weight.
- `Microsoft.Diagnostics.Tracing.TraceEvent` dependency (for EventPipe parsing used by `drhook_processes` + `drhook_snapshot`) moved into `DrHook.Engine` alongside the rest of the substrate.

## What 1.8.2 does NOT carry

- `drhook_step_test` is still wired (engine path returns a structured "not yet ported"). ADR-007 (Proposed 2026-05-23) removes the tool entirely in Phase 7 ‚ÄĒ the `VSTEST_HOST_DEBUG` trick is not getting ported; substrate-aligned child-process attach is the replacement (ADR-007 Phase 3).
- Stale `[Description]` strings on some MCP tools still reference netcoredbg ("Uses netcoredbg (MIT, DAP over stdio)"). ADR-007 Phase 7 cleans these.
- Cross-platform validation: only macOS/arm64 has been validated. `DrHook.Mcp` bundles all 7 RIDs (osx-arm64, osx-x64, linux-x64, linux-arm64, linux-musl-x64, win-x64, win-arm64) but no probe runs have been performed on any of them yet. ADR-007 Phase 9 is the time-budgeted cross-platform campaign.
- Teardown + concurrency hardening, test-runner debugging substrate, integration-test suite ‚ÄĒ all open. See ADR-007 (Proposed 2026-05-23) for the production-suitability sequencing.
