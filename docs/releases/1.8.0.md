# Sky Omega 1.8.0 — Cognitive Layers Entry Point

**Released:** 2026-05-17
**Tag:** `mercury-1.8.0`
**Predecessor:** [Mercury 1.7.74](#) — last 1.7.x release; ADR-043 metric emission decoupling

## What 1.8.0 means

**1.8.0 is the boundary between substrate work and cognitive work**, not a production-hardening release. The substrate-hardening arc completed across 1.7.x; 1.8.0 marks the entry point for the cognitive layers (Lucy / James / Mira / Sky) that the substrate carries.

This release does NOT bundle new substrate features. It bundles the *closure* of the substrate-hardening line and the *framing* for what cognitive-layer work begins next. The version bump exists to mark this milestone explicitly — the 1.7.x line is closed at 1.7.74; subsequent substrate work happens as 1.8.x releases alongside cognitive-layer work.

### Production readiness scope

Mercury 1.8.0 is **production-hardened for single-host workloads validated to 21.3 B triples** on a single M5 Max with 128 GB RAM. The substrate substrate has demonstrated 0 substrate failures across 8,564 unique query × substrate executions on the 1.7.x line.

What "production-hardened" means for 1.8.0:
- ✅ Substrate stability under load (cycle 10 r4 + truthy r1 + WGPB step C)
- ✅ Correctness (full W3C conformance + ADR-044 cross-format atom identity)
- ✅ Performance (85 h → 24 h on the same hardware across the Phase 6 → cycle 10 r4 arc; −71.8 %)
- ✅ Observability (ADR-035 metrics + ADR-043 bounded-staleness)
- ✅ Memory adaptivity (ADR-040 readahead + ADR-042 MPHF construction adapt to host RAM)
- ✅ Cleanup discipline (ADR-041 try/finally + 1.7.49 cleanup hook)

What's NOT in 1.8.0's production scope:
- ❌ Multi-host distributed deployment
- ❌ Public NuGet publication (tools installed via `./tools/install-tools.sh` from local source)
- ❌ Enterprise-grade SLAs / monitoring / signed releases
- ❌ DrHook engine BCL-only rewrite (deferred to 1.8.x — current netcoredbg-backed form is functionally sufficient for cognitive-layer bring-up)

## What begins in 1.8.x

The substrate work continues alongside cognitive layers. See the cognitive-layers roadmap (`docs/roadmap/cognitive-layers.md`) for the sequencing. High-level shape:

- **Lucy** — deep semantic memory layered on Mercury. The MCP semantic-memory pattern Sky Omega already uses through `mercury-mcp` becomes a proper layered substrate with provenance, consolidation cycles, and the EEE-discipline gates that MERCURY.md describes.
- **James** — orchestration with pedagogical guidance. Closes the `observability-discipline-systematic-not-reactive` limit structurally (periodic substrate audits as behavior, not as discipline). Resolves the meta-pattern of "we discover gaps by hitting them."
- **Mira** — surface/interaction layer. The presentation tier across CLI, chat, IDE extensions.
- **Sky** — agent surface integrating all three. The cognitive partner that the user already collaborates with informally through Claude Code becomes a proper persistent identity with continuity across sessions.

And substrate-side, parallel to cognitive work:

- **DrHook engine BCL-only rewrite** — first 1.8.x substrate-discipline task. Replaces netcoredbg + Microsoft.Diagnostics.NETCore.Client with a pure-BCL implementation per Sky Omega's substrate-independence principle. Resolves `project_drhook_eval_dead` and the `func-eval deadlock on macOS/ARM64` limit.
- **Minerva** continues BCL-only LLM inference development.
