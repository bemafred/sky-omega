Your AI coding agent can write the code and run the tests. What it can't do is watch the code run.

We spent ten weeks building the thing that lets it — and found that a debugger for an AI is a different instrument than the one in your IDE.

AI agents loop: compile, test, stop when the tests pass. But green tests only confirm what you expected. A deadlock, an infinite loop, a process pinned at 100% emit no signal a log-reading agent can see — so it reasons about runtime behavior from the only thing it can read: the source text. Exactly the move that produces confident, plausible, wrong code.

The fix is a third verb: compile, test, INSPECT. DrHook adds it — an agent attaches to a live .NET process, sets breakpoints, steps, reads variables at a stopped frame, and reports what it found instead of guessing.

—

The wrong answer that sounded right

Early on, one capability kept hanging. Asked why, the AI side of this collaboration gave a confident diagnosis: the platform is broken. Six citations, a clean causal chain — it read like diligence.

One question falsified it: does this work in any other debugger on the same machine? It did. The bug wasn't the platform's — it was local, and fixable. The lesson: a thorough-sounding answer with no falsifying test behind it isn't evidence, whoever produced it.

—

A debugger whose user isn't human

When the user is a reasoning loop, not a person at a screen, three things change:

• The tool names are an API — a backwards name isn't a style nit, it's a bug: the agent decides what to call by reading them.
• The program's own output becomes a protocol hazard, not a convenience.
• And the one no IDE has: DrHook makes the agent state a hypothesis before it looks — not "show me count" but "I expect count to be 3 here — show me." Then it shows what's actually there.

Every (expected, observed) pair becomes a record — a debugger that asks what you expect FIRST, because the gap between expected and observed is the signal the loop should learn from.

—

What "we" means

One human architect and one persistent AI collaborator — spare hours, shared discipline. When the external debugger we started on cost us control, we replaced it with our own: BCL-only. A dozen decision records, over eighty dated findings, all public. The wrong-answer story is the template: one side builds the elaborate answer, the other asks the falsifying question. Neither half does it alone.

Untrusted nothing is just absence. Checked nothing is evidence — Kjell Silverstein's poem, built into a debugger.

—
